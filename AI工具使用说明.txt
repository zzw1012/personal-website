AI工具使用说明
==============

1. 使用了哪些AI工具
Claude Code（Anthropic 官方 CLI 编程工具），通过子智能体驱动开发模式（Subagent-Driven Development）完成全部开发工作。

2. AI帮助完成了哪些工作
- 需求分析与方案设计：将测试题要求拆解为可执行的设计方案，输出完整的设计规格文档（技术选型对比、视觉设计方案、页面结构规划）
- 实施计划生成：将设计方案拆分为 11 个详细任务的实施计划，每个任务包含精确的代码、文件路径和验证标准
- 全部代码生成：生成完整的 HTML 结构、CSS 样式（CSS 变量体系、粒子动画背景、Hero 区排版、卡片布局、技能标签组、项目展示、联系方式、毛玻璃导航栏、渐入动画）和 JavaScript 交互代码（Canvas 粒子系统、Intersection Observer 滚动监听与导航高亮）
- 简历内容提取：通过 pdftotext 命令行工具从 PDF 简历中提取文本内容并转换为网站文案
- 部署指导：Git 初始化、GitHub Pages 配置与推送部署

3. 自己手动修改了哪些内容
- 技术方案的最终决策：选择方案 A（纯 HTML/CSS/JS）、确认现代科技风外壳 + 简洁内容卡片的视觉方向、选择 GitHub Pages 部署
- 设计方向的把控：深色背景 + 粒子动画 + 青蓝色强调色的科技感配色方案
- 技能标签分组结构的设计：将技能按语言/AI/后端/前端/工程五组分类
- GitHub 仓库创建：手动在 GitHub.com 创建仓库并启用 Pages
- 简历内容到网站文案的审核确认：所有文字内容均来自简历原文，未虚构

4. 遇到的问题，以及如何通过AI解决
- 简历 PDF 文件被 Claude Code 内置 Read 工具识别为"密码保护"无法读取 → AI 建议并通过 pdftotext 命令行工具直接提取文本内容
- pdftotext 默认编码导致中文字符变成方框/空白 → AI 建议添加 -enc UTF-8 参数解决中文编码问题
- pdftotext 使用 -layout 参数导致中文字符丢失 → AI 建议去掉 -layout 参数，使用默认布局模式，中文提取正常
- GitHub CLI（gh）未安装在本地环境 → AI 提供了浏览器手动创建仓库的替代方案
- GitHub Pages API 调用需要认证 Token → AI 提供了浏览器手动配置 Pages 的步骤
